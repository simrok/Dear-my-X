# 카카오톡 .txt 추출 가이드

이 문서는 두 부분으로 나뉩니다.

1. **사용자 가이드** — 앱 사용자가 카카오톡 대화를 .txt 로 내보내는 방법
2. **개발자 노트** — 파서가 지원하는 포맷 / 예시 / 제한사항

---

## 1. 사용자 가이드

### 모바일 (Android / iOS)

1. **카카오톡** 에서 대화방을 엽니다.
2. 우측 상단의 **메뉴(≡)** 를 누릅니다.
3. **대화방 설정** 으로 들어갑니다.
4. **대화 내용 내보내기** 를 선택합니다.
5. **"텍스트만 보내기"** 또는 **"텍스트로 저장"** 을 선택합니다.
6. 생성된 `.txt` 파일을 메일/드라이브 등으로 자기 자신에게 보내거나 폰에 저장합니다.
7. Dear my X 의 **대화 업로드** 화면에서 그 `.txt` 파일을 선택합니다.

### PC (Windows / macOS)

1. 카카오톡 PC 버전에서 대화방을 엽니다.
2. 대화창 우측 상단의 톱니바퀴(설정) 를 누릅니다.
3. **대화 내보내기 → 텍스트 파일(.txt)** 을 선택합니다.
4. 저장된 `.txt` 파일을 Dear my X 에 업로드합니다.

### 자주 묻는 질문

- **사진/이모티콘은 어떻게 처리되나요?** — 본문에서 자동으로 제외됩니다 ("사진", "이모티콘" 등 미디어-only 라인).
- **개인정보는 안전한가요?** — 업로드된 `.txt` 의 **전화번호 / 이메일 / 계좌 / 주민번호 / URL / 주소 추정 패턴** 은 임베딩 직전에 자동 마스킹됩니다(`[PHONE]`, `[EMAIL]` 등). 자세한 규칙은 개발자 노트 참고.
- **캡처 이미지로 올릴 수는 없나요?** — 현재 MVP 는 텍스트(.txt) 만 지원합니다. OCR 은 향후 검토 예정.

## 2. 개발자 노트

### 지원 포맷

| 포맷 | 우선순위 | 비고 |
| --- | --- | --- |
| **PC 내보내기 (Windows/macOS)** | ✅ 1순위 | `[발화자] [오후 3:01] 본문` |
| **모바일 내보내기 (Android/iOS)** | ✅ 2순위 (best-effort) | `2024년 1월 1일 오후 3:01, 발화자 : 본문` 또는 `2024. 1. 1. 오후 3:01, 발화자 : 본문` |
| 영문 / 다른 로케일 | ❌ | 한국어 ko_KR 가정 |

`KakaoParser.parse(text)` 는 두 형식 매칭 횟수 중 더 많은 쪽을 `detected_format` 으로 결정합니다.

### PC 형식 예시

```
KakaoTalk 대화, 닉네임 님과 카톡대화
저장한 날짜 : 2024년 1월 1일 오후 3:00

--------------- 2024년 1월 1일 월요일 ---------------
[홍길동] [오후 3:01] 안녕!
[김철수] [오후 3:02] 어 반가워
[김철수] [오후 3:02] 오늘 뭐해?
                  여러 줄로 이어서 쓰면 직전 메시지 본문에 합쳐짐
[홍길동] [오후 3:05] 영화 보러 가자
홍길동님이 들어왔습니다.
사진
[홍길동] [오후 3:10] 010-1234-5678 로 연락해
```

위 입력에서 추출되는 결과:

```python
ParseResult(
  messages=[
    KakaoMessage('홍길동', 2024-01-01 15:01, '안녕!'),
    KakaoMessage('김철수', 2024-01-01 15:02, '어 반가워'),
    KakaoMessage('김철수', 2024-01-01 15:02,
                 '오늘 뭐해?\n여러 줄로 이어서 쓰면 직전 메시지 본문에 합쳐짐'),
    KakaoMessage('홍길동', 2024-01-01 15:05, '영화 보러 가자'),
    KakaoMessage('홍길동', 2024-01-01 15:10, '010-1234-5678 로 연락해'),
  ],
  speakers=['홍길동', '김철수'],
  detected_format='pc',
  skipped_lines=...,  # 헤더/시스템/사진 라인
)
```

마스킹 후 마지막 메시지 본문은 `[PHONE] 로 연락해` 가 됩니다.

### 모바일 형식 예시

```
닉네임 님과의 카카오톡 대화
저장한 날짜 : 2024-01-01 15:00:00

2024년 1월 1일 오후 3:01, 홍길동 : 안녕!
2024년 1월 1일 오후 3:02, 김철수 : 어 반가워
```

### 정규식 (참고)

| 대상 | regex |
| --- | --- |
| PC 날짜 구분선 | `^-+\s*(\d{4})년\s*(\d{1,2})월\s*(\d{1,2})일.*-+\s*$` |
| PC 메시지 라인 | `^\[(.+?)\]\s*\[(오전\|오후)\s*(\d{1,2}):(\d{2})\]\s*(.*)$` |
| 모바일 메시지 | `^(\d{4})[년.\s]+(\d{1,2})[월.\s]+(\d{1,2})[일.\s]+(오전\|오후)\s*(\d{1,2}):(\d{2}),\s*([^:]+?)\s*:\s*(.*)$` |
| 시스템 메시지 | "님이 들어왔습니다." / "님이 나갔습니다." / "님을 초대했습니다." / "채팅방을 만들었습니다." 등 |
| 미디어 only | `^(사진\|사진 \d+장\|동영상\|이모티콘\|음성메시지\|파일\|<.+>)$` |

### 멀티라인 메시지 처리

직전 라인이 메시지 형식 매칭에 성공한 상태에서 빈 줄/시스템 라인/날짜 구분선이 나오기 전까지의
모든 라인은 직전 메시지 본문에 `\n` 으로 이어붙여집니다. 빈 줄을 만나면 멀티라인 컨텍스트가 종료됩니다.

### 알려진 제한 사항

- **편집/삭제된 메시지** 표시(예: `이 메시지는 삭제되었습니다.`) 는 일반 메시지로 그대로 들어옵니다 (TODO: 별도 필터).
- **닉네임에 `[`, `]` 가 포함** 된 경우 PC 형식 정규식이 잘못 매칭될 수 있습니다.
- **시간대(timezone)** 는 부여하지 않습니다. naive `datetime` 사용.
- **연도가 없는 모바일 export** 형식은 미지원 (모바일 일부 OS 는 첫 줄에 연도가 없음).
- **파일 인코딩** 은 UTF-8(BOM 포함) 우선, 실패 시 CP949 로 fallback. 다른 인코딩은 422 에러.
